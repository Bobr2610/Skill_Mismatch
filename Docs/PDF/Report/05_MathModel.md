# 5. Математическая модель оценки динамики компетенций

## 5.1. Мотивация модели

Статическая оценка компетенций (однократный замер) не отражает **динамику** профессионального развития. Навыки имеют конечный «срок годности»:

- Знание конкретного фреймворка устаревает за 2–3 года
- Навыки проектирования архитектуры сохраняют актуальность 5–7 лет
- Фундаментальные знания практически не устаревают

Необходима модель, которая учитывает:

1. **Накопление** — новые достижения увеличивают оценку
2. **Затухание** — без подтверждения навыки «обесцениваются»
3. **Зависимость от роли** — скорость затухания различается для разных позиций

## 5.2. Формула рекуррентного пересчёта

Оценка пересчитывается по формуле:

$$P_{new} = P_{current} \times K + M$$

Где:
- $P_{new}$ — новое значение параметра (0–100)
- $P_{current}$ — текущее значение параметра
- $K$ — коэффициент затухания, зависящий от позиции
- $M$ — свежий вклад за период

Результат ограничивается диапазоном $[0, 100]$:

$$P_{new} = \min(100, \max(0, P_{current} \times K + M))$$

## 5.3. Коэффициенты затухания по позициям

| Позиция | K | Равновесное значение (при M=30) | Интерпретация |
|---------|---|-------------------------------|---------------|
| Начинающий | 0.4 | 50 | Быстрое затухание |
| Средний | 0.5 | 60 | Умеренное затухание |
| Старший | 0.6 | 75 | Медленное затухание |
| Эксперт | 0.7 | 100 | Очень медленное затухание |
| Руководитель | 0.75 | 100 | Минимальное затухание |

## 5.4. Анализ равновесия

При постоянном ежемесячном вкладе M система стремится к равновесному значению:

$$P^* = \frac{M}{1 - K}$$

Это позволяет определять минимальные требования для каждой позиции.

## 5.5. Динамика затухания

При прекращении активной работы параметр затухает экспоненциально:

$$P(t) = P_0 \times K^t$$

Время снижения до 50% от начального значения:

$$t_{1/2} = \frac{\ln 0.5}{\ln K}$$

| Позиция | K | t₁/₂ (месяцев) |
|---------|---|-----------------|
| Начинающий | 0.4 | 0.76 |
| Средний | 0.5 | 1.0 |
| Старший | 0.6 | 1.36 |
| Эксперт | 0.7 | 1.94 |
| Руководитель | 0.75 | 2.41 |

## 5.6. Обнаружение несоответствия позиции

Модель позволяет выявить несоответствие по паттерну:

1. До повышения: параметры стабильно растут — сигнал к повышению
2. После повышения: требования к вкладу растут
3. При несоответствии: параметры начинают устойчиво снижаться

Три последовательных периода снижения после повышения — триггер для пересмотра позиции.
