# 6. Математическая модель: функция затухания Грина

## 6.1. Мотивация модели

Статическая оценка компетенций (однократный замер) не отражает **динамику** профессионального развития. В ИТ-индустрии навыки имеют конечный «срок годности»:

- Знание конкретного фреймворка устаревает за 2–3 года
- Навыки проектирования архитектуры сохраняют актуальность 5–7 лет
- Фундаментальные алгоритмические знания практически не устаревают

Необходима модель, которая учитывает:

1. **Накопление** — новые достижения увеличивают оценку
2. **Затухание** — без подтверждения, навыки «обесцениваются»
3. **Зависимость от роли** — скорость затухания различается для разных грейдов

## 6.2. Формула рекуррентного пересчёта

Для каждого из 6 параметров оценка пересчитывается ежемесячно по формуле:

$$
P_{new} = P_{current} \times K(role) + M
$$

Где:

- $P_{new}$ — новое значение параметра (0–100)
- $P_{current}$ — текущее значение параметра
- $K(role)$ — коэффициент затухания, зависящий от грейда
- $M$ — свежий вклад за месяц (monthly value), вычисленный по данным коммитов

Результат ограничивается диапазоном $[0, 100]$:

$$P_{new} = \min(100, \max(0, P_{current} \times K + M))$$

## 6.3. Коэффициенты затухания по ролям

| Роль | $K$ | Равновесное значение (при $M = 30$) | Интерпретация |
|---|---|---|---|
| **Junior** | 0.4 | 50 | Быстрое затухание: без активной работы навыки теряются |
| **Middle** | 0.5 | 60 | Умеренное затухание: базовые навыки сохраняются дольше |
| **Senior** | 0.6 | 75 | Медленное затухание: глубокие знания устойчивы |
| **Staff** | 0.7 | 100 | Очень медленное затухание: экспертиза сохраняется |
| **Lead** | 0.75 | 120 → cap 100 | Минимальное затухание: стратегические навыки стабильны |

## 6.4. Анализ равновесия

При постоянном ежемесячном вкладе $M$ система стремится к **равновесному** (стационарному) значению $P^*$:

$$P^* = P^* \times K + M \implies P^* = \frac{M}{1 - K}$$

| $K$ | $P^*$ при $M = 30$ | $P^*$ при $M = 50$ |
|---|---|---|
| 0.4 | 50 | 83 |
| 0.5 | 60 | 100 |
| 0.6 | 75 | 125 → 100 |
| 0.7 | 100 | 167 → 100 |
| 0.75 | 120 → 100 | 200 → 100 |

Это означает, что:

- **Junior** при стабильном вкладе $M = 30$ выходит на уровень 50 — система сигнализирует о необходимости роста
- **Senior** при том же вкладе достигает 75 — для поддержания статуса нужен больший вклад
- **Lead** при $M = 30$ «упирается» в потолок 100 — подтверждение соответствия грейду

## 6.5. Динамика затухания без вклада

При прекращении активной работы ($M = 0$) параметр затухает экспоненциально:

$$P(t) = P_0 \times K^t$$

где $t$ — номер месяца. Время полураспада (снижение до 50% от начального):

$$t_{1/2} = \frac{\ln 0.5}{\ln K}$$

| Роль | $K$ | $t_{1/2}$ (мес.) |
|---|---|---|
| Junior | 0.4 | 0.76 |
| Middle | 0.5 | 1.0 |
| Senior | 0.6 | 1.36 |
| Staff | 0.7 | 1.94 |
| Lead | 0.75 | 2.41 |

У Junior'а оценка падает вдвое менее чем за месяц — это стимулирует непрерывное обучение. У Lead'а — примерно за 2.4 месяца, что отражает устойчивость экспертных знаний.

## 6.6. Обнаружение Принципа Питера через модель

Модель позволяет выявить Принцип Питера по характерному паттерну:

1. **До повышения**: параметры стабильно растут, $P > P^*_{role}$ — сигнал к повышению
2. **Сразу после повышения**: $K$ увеличивается (новый грейд), требования к $M$ растут
3. **Если Принцип Питера сработал**: сотрудник не может обеспечить необходимый $M$ на новой позиции, и $P$ начинает устойчиво снижаться

Формально:

$$\text{Peter Principle detected} \iff \forall t > t_{promote}: \quad P(t) < P(t-1) \text{ для } t = t_{promote}+1, \ldots, t_{promote}+3$$

Три последовательных месяца снижения после повышения — триггер для пересмотра грейда или целевого обучения.
