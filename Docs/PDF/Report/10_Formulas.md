# 10. Методы и формулы оценки производительности

## Цель раздела

Настоящий раздел представляет собой справочно-методологическое пособие по математическому аппарату оценки трудовой производительности в контексте диагностики Skill Mismatch. Цель — систематизация формул, нормативных порогов и алгоритмов интерпретации результатов, применимых для количественного измерения соответствия компетенций сотрудников требованиям занимаемых должностей. Приведённые методики охватывают индивидуальную, командную и организационную оценку, отраслевые спецификации, динамику затухания навыков и финансовые аспекты управления человеческим капиталом. Формулы раздела позволяют формализовать разрывы между номинальной и фактической производительностью и перевести качественное наблюдение в измеримый сигнал.

## 10.1. Фундаментальные формулы производительности

### 10.1.1. Базовое определение производительности

Производительность труда определяется как отношение объёма произведённых рабочих эффектов к затраченным трудовым ресурсам:

$$W = \frac{E}{N}$$

где:
- $W$ — производительность труда (output per unit of labor input)
- $E$ — рабочие эффекты (units produced, tasks completed, revenue generated)
- $N$ — трудозатраты (hours worked, person-days, FTE)

**Единицы измерения:** задачи/час, строки кода/день, единицы продукции/смена, рубли/человеко-час.

### 10.1.2. Трудоёмкость (обратный показатель)

$$L = \frac{N}{E}$$

Трудоёмкость показывает количество трудовых затрат, необходимых для производства единицы продукции. Снижение $L$ во времени является индикатором роста эффективности.

*Пример расчёта.* При $E = 240$ задач в месяц и $N = 160$ человеко-часов: $W = 240 / 160 = 1{,}5$ задачи/час; $L = 160 / 240 = 0{,}67$ часа/задачу.

### 10.1.3. Выручка на сотрудника (Revenue per Employee, RPE)

$$RPE = \frac{\text{Total Revenue}}{\text{Number of Employees}}$$

**Бенчмарки (2024–2025):**

| Сегмент | Медиана RPE | Верхний квартиль |
|---|---|---|
| SaaS (частные компании) | $175K–$200K | $300K–$500K |
| SaaS (публичные, developer tools) | >$400K | >$600K |
| Производство | $150K–$250K | $350K–$450K |
| Розничная торговля | $100K–$180K | $250K–$350K |

### 10.1.4. Индекс производительности (Productivity Index)

$$PI = \frac{P_{\text{current}}}{P_{\text{base}}} \times 100\%$$

где $P_{\text{current}}$ — текущее значение производительности, $P_{\text{base}}$ — базовое (эталонное) значение.

**Интерпретация:** $PI > 100\%$ — рост производительности; $PI = 100\%$ — стабильность; $PI < 100\%$ — снижение.

*Пример расчёта.* Базовое $W_0 = 0{,}075$ задачи/час, текущее $W_1 = 0{,}083$ задачи/час: $PI = 0{,}083 / 0{,}075 \times 100\% = 111\%$ (прирост 11%).

**Сигнал Skill Mismatch:** падение $PI$ ниже 95% при стабильном составе команды и неизменных бизнес-процессах указывает на накопление несоответствия компетенций.

## 10.2. Индивидуальная оценка

### 10.2.1. Индивидуальная производительность по выработке

$$P_{\text{individual}} = \frac{\text{Units Produced}}{\text{Hours Worked}}$$

Применима при квантуемых результатах: операторы контакт-центров, сборщики, специалисты по обработке данных, линейный производственный персонал.

### 10.2.2. Коэффициент эффективности

$$\text{Efficiency} = \frac{T_{\text{standard}}}{T_{\text{actual}}} \times 100\%$$

где $T_{\text{standard}}$ — нормативное время выполнения, $T_{\text{actual}}$ — фактическое время выполнения.

**Нормативные зоны:** $\text{Efficiency} \geq 100\%$ — работа быстрее норматива; $85\% \leq \text{Efficiency} < 100\%$ — допустимое отклонение; $\text{Efficiency} < 85\%$ — критическое отклонение.

*Пример расчёта.* Норматив обработки заявки $T_{\text{standard}} = 20$ мин, фактическое время $T_{\text{actual}} = 25$ мин: $\text{Efficiency} = 20/25 \times 100\% = 80\%$. Результат ниже порога 85% — сотрудник квалифицируется как кандидат на переобучение либо перераспределение задач.

### 10.2.3. Стандартизированная оценка (z-score)

$$z = \frac{X - \mu}{\sigma}$$

где $X$ — показатель оцениваемого сотрудника, $\mu$ — среднее значение по группе сравнения, $\sigma$ — стандартное отклонение группы.

**Статистические пороги:** $|z| \leq 1$ — в пределах 68% распределения (норма); $1 < |z| \leq 2$ — отклонение в пределах 95% (внимание); $|z| > 2$ — за пределами 95% (анализ причин); $|z| > 3$ — за пределами 99{,}7% (критическое отклонение).

*Пример расчёта.* Показатель сотрудника $X = 45$ задач/мес, среднее по группе $\mu = 60$ задач/мес, стандартное отклонение $\sigma = 10$ задач/мес: $z = (45 - 60) / 10 = -1{,}5$. Значение $z = -1{,}5$ соответствует нижним ~7% распределения — основание для детального анализа причин отставания.

### 10.3.1. Командная производительность с коррекцией на качество

$$P_{\text{team}} = \frac{E_{\text{completed}} - E_{\text{defective}}}{N_{\text{total}}}$$

Вычитание дефектных результатов предотвращает инфляцию показателей за счёт жертвования качеством («производительность ради количества»).

*Пример расчёта.* Выполнено задач $E_{\text{completed}} = 200$, возвращено на доработку $E_{\text{defective}} = 30$, затраты $N_{\text{total}} = 800$ человеко-часов: $P_{\text{team}} = (200 - 30) / 800 = 170 / 800 = 0{,}2125$ задачи/час. Без коррекции: $200/800 = 0{,}25$ задачи/час (завышение на 17{,}6%).

### 10.3.2. Коэффициент завершения задач в срок (Task Completion Rate)

$$TCR = \frac{\text{Tasks Completed On Time}}{\text{Total Assigned Tasks}} \times 100\%$$

**Нормативы:** $TCR \geq 85\%$ — зрелая команда; $70\% \leq TCR < 85\%$ — требуется оптимизация; $TCR < 70\%$ — системные проблемы планирования либо компетенций.

### 10.3.3. Индексы выполнения расписания и бюджета

**Schedule Performance Index:**
$$SPI = \frac{EV}{PV}$$

**Cost Performance Index:**
$$CPI = \frac{EV}{AC}$$

где $EV$ (Earned Value) — бюджетная стоимость выполненных работ, $PV$ (Planned Value) — бюджетная стоимость запланированных работ, $AC$ (Actual Cost) — фактические затраты.

**Интерпретация:** $SPI, CPI > 1$ — опережение графика/экономия бюджета; $SPI, CPI = 1$ — соответствие плану; $SPI, CPI < 1$ — отставание/перерасход.

*Пример расчёта.* В проекте на контрольной дате: $PV = 4{,}0$ млн руб. (план), $EV = 3{,}2$ млн руб. (фактически выполнено), $AC = 3{,}6$ млн руб. (фактические затраты). Тогда $SPI = 3{,}2 / 4{,}0 = 0{,}80$ (отставание 20%), $CPI = 3{,}2 / 3{,}6 = 0{,}89$ (перерасход 11%). Комбинация $SPI < 1$ и $CPI < 1$ — классический паттерн команды с дефицитом компетенций относительно задач.

## 10.4. Отраслевые формулы

### 10.4.1. DORA-метрики (ИТ-сфера)

DORA-метрики (DevOps Research and Assessment) — отраслевой стандарт оценки инженерных команд.

| Метрика | Формула | Описание |
|---|---|---|
| Deployment Frequency (DF) | $N_{\text{deploys}} / T$ | Частота успешных релизов |
| Lead Time for Changes (LT) | $\text{median}(t_{\text{deploy}} - t_{\text{commit}})$ | Время от коммита до продакшена |
| Change Failure Rate (CFR) | $\text{Failed Deploys} / \text{Total Deploys} \times 100\%$ | Доля неудачных изменений |
| Mean Time to Restore (MTTR) | $\sum \text{Downtime} / N_{\text{incidents}}$ | Среднее время восстановления |

#### Уровни производительности DORA

| Уровень | Deployment Frequency | Lead Time | CFR | MTTR | Доля команд (2025) |
|---|---|---|---|---|---|
| **Elite** | По требованию (многократно в день) | $< 1$ час | 0–5% | $< 1$ час | ~18% |
| **High** | Раз в неделю — раз в месяц | 1 день – 1 неделя | 6–15% | $< 1$ дня | ~28% |
| **Medium** | Раз в месяц — раз в полгода | 1 неделя – 1 месяц | 16–30% | 1 день – 1 неделя | ~32% |
| **Low** | Реже раза в полгода | $> 1$ месяца | $> 30\%$ | $> 1$ недели | ~22% |

*Пример диагностики Skill Mismatch.* Команда из 8 инженеров (2 Senior, 4 Middle, 2 Junior) показывает Deployment Frequency = 1 раз в месяц (Medium), Lead Time = 3 недели (Low), CFR = 35% (Low), MTTR = 4 дня (Medium). По всем четырём метрикам — уровень Low или Medium. При номинальном грейдовом составе, соответствующем High/Elite, имеет место системное несоответствие: либо распределение задач не соответствует компетенциям, либо грейды завышены.

### 10.4.2. Сфера продаж

**Выручка на менеджера:**
$$\text{Revenue per Rep} = \frac{\text{Total Revenue}}{N_{\text{reps}}}$$

**Скорость продаж (Sales Velocity):**
$$SV = \frac{N_{\text{opportunities}} \times \overline{D} \times W_r}{L_c}$$

где $N_{\text{opportunities}}$ — число сделок в воронке, $\overline{D}$ — средний чек (Average Deal Size), $W_r$ — win rate (доля выигранных сделок), $L_c$ — средняя длина цикла продаж (дни).

*Пример расчёта.* $N_{\text{opportunities}} = 50$ сделок, $\overline{D} = 200{,}000$ руб., $W_r = 0{,}25$ (25%), $L_c = 30$ дней: $SV = (50 \times 200{,}000 \times 0{,}25) / 30 = 2{,}500{,}000 / 30 \approx 83{,}333$ руб./день выручки, генерируемой воронкой.

**Конверсия лидов:**
$$LCR = \frac{N_{\text{converted}}}{N_{\text{leads}}} \times 100\%$$

### 10.4.3. Производство

**Общая эффективность оборудования (Overall Equipment Effectiveness):**
$$OEE = A \times P \times Q$$

где $A = T_{\text{run}} / T_{\text{planned}}$ — доступность (Availability), $P = Q_{\text{actual}} / Q_{\text{theoretical}}$ — производительность (Performance), $Q = Q_{\text{good}} / Q_{\text{total}}$ — качество (Quality).

**Уровни OEE:**

| Уровень | OEE | Частота в отрасли |
|---|---|---|
| World-class | $\geq 85\%$ | ~6% предприятий |
| Приемлемый (high-mix) | 75–80% | ~25% предприятий |
| Средний | 55–60% | большинство |
| Требует улучшений | $< 60\%$ | ~25% |

Бенчмарк 85% выведен для высокообъёмных линий; для high-mix производства целевое значение составляет 75–80%.

*Пример расчёта.* Линия: $A = 0{,}90$ (доступность), $P = 0{,}85$ (производительность), $Q = 0{,}95$ (качество). Тогда $OEE = 0{,}90 \times 0{,}85 \times 0{,}95 = 0{,}727$ (72{,}7%). Декомпозиция позволяет локализовать источник потерь: в данном случае основной резерв — в $P$-факторе (скоростные потери).

## 10.5. Затухание навыков

### 10.5.1. Экспоненциальная модель затухания

$$S(t) = S_0 \cdot e^{-\lambda t} = S_0 \cdot (1 - r)^t$$

где $S(t)$ — уровень навыка в момент времени $t$, $S_0$ — исходный уровень навыка, $\lambda$ — коэффициент затухания (непрерывный), $r$ — дискретный коэффициент утраты за единицу времени, $t$ — время без практики.

### 10.5.2. Период полураспада навыка

$$t_{1/2} = \frac{\ln 2}{\lambda} \approx \frac{0{,}693}{\lambda}$$

### 10.5.3. Эмпирические коэффициенты затухания

| Тип навыка | $r$ (в день) | $t_{1/2}$ | Примеры |
|---|---|---|---|
| Моторные | 0{,}01 | ~69 дней | операторские навыки, пайка |
| Когнитивные (обычные) | 0{,}02 | ~34 дня | SQL, формулы Excel |
| Быстрозатухающие | 0{,}03 | ~23 дня | API новой версии, фреймворки |

*Пример расчёта.* Для когнитивного навыка ($r = 0{,}02$) через 30 дней: $S(30) = S_0 \cdot (1 - 0{,}02)^{30} = S_0 \cdot 0{,}98^{30} \approx S_0 \cdot 0{,}545$ (сохраняется ~55% исходного уровня). Через 90 дней: $S(90) = S_0 \cdot 0{,}98^{90} \approx S_0 \cdot 0{,}16$ (сохраняется ~16%).

**Связь со Skill Mismatch.** Модель затухания описывает эффект «выпадения» из компетенции: сотрудник, формально обученный 2 года назад и не применявший навык, фактически обладает долей $e^{-\lambda \cdot 730}$ от исходной компетентности. При планировании оценки необходимо различать актуальную и исторически подтверждённую квалификацию.

## 10.6. Комплексные модели оценки

### 10.6.1. Взвешенное оценивание (Weighted Scoring)

$$WS = \sum_{i=1}^{n} w_i \cdot s_i, \qquad \sum_{i=1}^{n} w_i = 1$$

где $w_i$ — вес $i$-го критерия, $s_i$ — балл по $i$-му критерию (обычно 0–100).

*Пример оценки менеджера по продажам:*

| Критерий | Вес $w_i$ | Балл $s_i$ | $w_i \cdot s_i$ |
|---|---|---|---|
| Выполнение плана | 0{,}35 | 85 | 29{,}75 |
| Новые клиенты | 0{,}25 | 70 | 17{,}50 |
| Качество работы | 0{,}20 | 90 | 18{,}00 |
| Командная работа | 0{,}10 | 80 | 8{,}00 |
| Развитие | 0{,}10 | 75 | 7{,}50 |
| **Итого** | **1{,}00** | — | **80{,}75** |

### 10.6.2. Модель 360-градусной оценки

$$\text{Score}_{360} = \frac{w_S \cdot S + w_P \cdot P + w_{\text{Sub}} \cdot \text{Sub} + w_C \cdot C + w_{\text{Self}} \cdot \text{Self}}{w_S + w_P + w_{\text{Sub}} + w_C + w_{\text{Self}}}$$

где $S$ — оценка руководителя, $P$ — коллег, $\text{Sub}$ — подчинённых, $C$ — клиентов, $\text{Self}$ — самооценка.

**Типовые веса:** руководитель $w_S \in [0{,}30; 0{,}40]$; коллеги $w_P \in [0{,}20; 0{,}25]$; подчинённые $w_{\text{Sub}} \in [0{,}10; 0{,}15]$; клиенты $w_C \in [0{,}15; 0{,}20]$; самооценка $w_{\text{Self}} = 0{,}10$.

### 10.6.3. Рекуррентная модель обновления оценки

$$P_{\text{new}} = P_{\text{current}} \cdot K + M$$

где $P_{\text{new}}$ — обновлённая оценка, $P_{\text{current}}$ — текущая оценка, $K \in [0; 1]$ — коэффициент инерции (доля, в которой новая оценка опирается на текущую), $M$ — поправка по результатам очередного периода.

*Пример расчёта.* При $K = 0{,}7$, $M = +5$, $P_{\text{current}} = 70$: $P_{\text{new}} = 70 \cdot 0{,}7 + 5 = 49 + 5 = 54$. При отрицательной динамике ($M = -3$): $P_{\text{new}} = 70 \cdot 0{,}7 - 3 = 46$. Модель обеспечивает сглаживание шума и реакцию на устойчивые тренды — в том числе на прогрессирующее снижение производительности, характерное для Skill Mismatch.

## 10.7. Финансовые метрики HR

### 10.7.1. Возврат инвестиций в персонал (ROI)

$$ROI = \frac{\text{Net Benefits} - \text{Costs}}{\text{Costs}} \times 100\%$$

*Пример расчёта.* Программа обучения стоимостью $\$100{,}000$ принесла дополнительный доход $\$250{,}000$. Чистая выгода $\$150{,}000$. Тогда $ROI = (150{,}000 / 100{,}000) \times 100\% = 150\%$.

### 10.7.2. Рентабельность человеческого капитала (HCROI)

$$HCROI = \frac{\text{Revenue} - \text{Operating Expenses (excl. comp.)}}{\text{Total Compensation}}$$

**Интерпретация:** показывает, сколько единиц маржи генерируется на каждую единицу затрат на оплату труда.

*Пример расчёта.* Выручка 10{,}000{,}000 руб., операционные расходы (без ФОТ) 6{,}000{,}000 руб., ФОТ 2{,}000{,}000 руб. Тогда $HCROI = (10{,}000{,}000 - 6{,}000{,}000) / 2{,}000{,}000 = 4{,}000{,}000 / 2{,}000{,}000 = 2{,}0$. На каждый рубль ФОТ приходится 2 рубля маржинального дохода.

### 10.7.3. Стоимость текучести кадров

$$\text{Cost}_{\text{turnover}} = N_{\text{separations}} \times \overline{C}_{\text{sep}}$$

где $N_{\text{separations}}$ — количество увольнений, $\overline{C}_{\text{sep}}$ — средняя стоимость замены одного сотрудника.

**Средняя стоимость замены (% годового оклада):**

| Категория | $\overline{C}_{\text{sep}}$ |
|---|---|
| Frontline / линейный персонал | ~40% |
| Технические / профессиональные роли | ~80% |
| Руководители и менеджеры | ~150–200% |

*Пример расчёта.* Компания с 500 сотрудниками, текучесть 15%, средний оклад 1{,}200{,}000 руб./год. $N_{\text{separations}} = 500 \cdot 0{,}15 = 75$ человек. Средняя $\overline{C}_{\text{sep}} = 80\%$ (смешанный состав). $\overline{C}_{\text{sep}} = 1{,}200{,}000 \cdot 0{,}80 = 960{,}000$ руб. Тогда $\text{Cost}_{\text{turnover}} = 75 \cdot 960{,}000 = 72{,}000{,}000$ руб./год.

**Связь со Skill Mismatch.** Стоимость замены применяется как основа сравнения издержек «уволить несоответствующего сотрудника» против «переобучить» — параметр, критичный для рекомендательного модуля системы.

| 2 | Сравнение сотрудника с когортой грейда | $z$-score, $WS$ | $z \in [-1; 1]$; $WS \geq 70$ | $z < -1{,}5$; $WS < 60$ | Несоответствие $z$-профиля ожидаемому для грейда |
| 3 | Командная оценка в проектной деятельности | $P_{\text{team}}$, $TCR$, $SPI$, $CPI$ | $TCR \geq 85\%$; $SPI, CPI \geq 0{,}95$ | $TCR < 70\%$; $SPI, CPI < 0{,}85$ | Комбинация $SPI < 1$ и $CPI < 1$ |
| 4 | Оценка инженерной ИТ-команды | DORA (DF, LT, CFR, MTTR), Cycle Time | Уровень High/Elite DORA | Уровень Low DORA | Грейдовый состав $\geq$ Middle при метриках Low |
| 5 | Оценка менеджеров по продажам | Revenue per Rep, $SV$, $LCR$, $WS$ | $LCR \geq$ отрасл. медианы; $SV$ растёт | $LCR < 0{,}5$ медианы; $SV$ падает | $W_r$ падает $> 2$ кварталов подряд |
| 6 | Оценка производственной линии / оператора | $OEE$, Labor Intensity, $Q$-фактор | $OEE \geq 75\%$ (high-mix), $\geq 85\%$ (high-volume) | $OEE < 60\%$ | Падение $P$-фактора OEE при стабильном $A$ |
| 7 | Оценка управленческих компетенций | 360$^\circ$, ROI инициатив руководителя, удержание команды | $\text{Score}_{360} \geq 75$; текучесть $< 10\%$ | $\text{Score}_{360} < 60$; текучесть $> 20\%$ | Паттерн Peter principle |
| 8 | Диагностика затухания навыков | $S(t) = S_0 \cdot e^{-\lambda t}$, тесты повторной аттестации | $S(t) / S_0 \geq 0{,}75$ | $S(t) / S_0 < 0{,}5$ | Формальная квалификация при $S(t)/S_0 < 0{,}5$ |
| 9 | Финансовая оценка HR-инициатив | $ROI$, $HCROI$, $\text{Cost}_{\text{turnover}}$ | $ROI > 0\%$; $HCROI \geq 2$ | $ROI < 0\%$; $HCROI < 1$ | Рост $\text{Cost}_{\text{turnover}}$ при стабильной численности |
| 10 | Интегральная оценка сотрудника в динамике | $PI$, рекуррентная $P_{\text{new}} = P_{\text{current}} \cdot K + M$ | $PI \geq 100\%$ | $PI < 90\%$ два периода подряд | Устойчивое $M < 0$ при неизменных задачах |

**Правила применения матрицы:**
1. Для надёжной диагностики Skill Mismatch необходимо комбинировать не менее трёх строк матрицы: индивидуальная метрика (строки 1–2), контекстная отраслевая (4–6) и динамическая (10).
2. Одиночное пересечение порога «проблема» по одной строке — повод для уточнения; пересечение по двум и более — основание для формальной диагностики несоответствия.
3. Пороговые значения калибруются под отраслевую медиану: абсолютные значения различаются между отраслями, тогда как относительные (z-score, $PI$, $SPI$) сохраняют интерпретируемость.

## 10.9. Принципы построения системы оценки

Корректная система оценки, предназначенная для выявления Skill Mismatch, строится на пяти принципах:

**Принцип 1: Многомерность.** Использование не менее 3–5 метрик из разных категорий матрицы 10.8. Одномерная оценка (например, только выработка) систематически недооценивает качество и переоценивает скорость.

**Принцип 2: Баланс количественного и качественного.** Количественные метрики (DORA, KPI, OEE) сочетаются с качественными (360$^\circ$, экспертное ревью).

**Принцип 3: Своевременность.** Ежегодные ревью не фиксируют затухание навыков (раздел 10.5) и тренды $PI$. Оптимальная частота — ежемесячная либо квартальная.

**Принцип 4: Стандартизация.** Приведение разнородных показателей к единой шкале (z-score, нормированные баллы 0–100) — условие сравнимости и агрегации.

**Принцип 5: Прозрачность.** Критерии и пороговые значения публикуются; сотрудник должен иметь возможность воспроизвести расчёт собственной оценки.

Ключевое утверждение раздела: выявление несоответствия компетенций — не задача выбора «лучшей» формулы, а задача корректного сопоставления цели, контекста и набора взаимодополняющих метрик с нормативными порогами, калиброванными под отрасль.*
