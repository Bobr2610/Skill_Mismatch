# 7. Практическая реализация: платформа DevMetrics

## 7.1. Архитектура системы

Платформа DevMetrics реализует описанную математическую модель в виде веб-приложения с модульной архитектурой:

![Архитектура платформы DevMetrics](Report/7_1.png){ width=85% }

### Технологический стек

| Компонент | Технология | Обоснование |
|---|---|---|
| **Backend** | Python Flask | Лёгкий, быстрый в разработке, идеален для прототипа |
| **Database** | SQLite | Нулевая конфигурация, подходит для MVP |
| **Frontend** | Vanilla JS + Tailwind CSS | Без build-step, мгновенный деплой |
| **AI Engine** | OpenRouter (GLM 4.5 Air) | Бесплатный API, достаточное качество анализа |
| **Data Source** | GitHub API | Реальные коммиты и PR для анализа |
| **Deployment** | Docker Compose | Одна команда для запуска всей системы |

## 7.2. Ключевые модули

### Модуль управления сотрудниками

- CRUD-операции с профилями
- Хранение 6 параметров и истории пересчётов
- Привязка к GitHub-аккаунту для реального анализа

### Модуль ИИ-анализа (ai.py)

Два режима работы:

**Режим 1 — Анализ коммитов:**

```
Коммит → AI (LLM) → Инкременты (0-5) по 6 параметрам → Накопление
```

ИИ получает сообщение коммита, diff-статистику, и возвращает оценку вклада в каждый из 6 параметров.

**Режим 2 — Карьерная рекомендация:**

```
Профиль (6 параметров) + Роль → AI → {promote | keep | demote} + обоснование
```

ИИ сравнивает профиль с эталонными требованиями и формирует аргументированную рекомендацию.

### Модуль Gorner`s Principle (пересчёт по формуле Горнера)

Реализует формулу $P_{new} = P_{current} \times K + M$:

- Автоматический ежемесячный пересчёт для всех сотрудников
- Сохранение истории (before/after/monthly\_value/K) в таблице `stats_history`
- Визуализация динамики на графиках профиля

### Dashboard с реальными данными GitHub

- **KPI в реальном времени**: количество коммитов, PR, cycle time
- **Лента активности**: последние коммиты из подключённого репозитория
- **Командная статистика**: агрегированные показатели команды

## 7.3. API платформы

Базовый путь для всех запросов: `/api/employees`.

**Режим 1 — Коммиты и затухание:**

- `POST .../commit-contribution` — ИИ-анализ коммита, начисление баллов
- `POST .../monthly-decay` — применение месячного затухания
- `POST /monthly-decay-all` — затухание для всех сотрудников

**Режим 2 — Карьерная рекомендация:**

- `POST .../recommendation` — рекомендация promote / keep / demote

**Пересчёт по Грину:**

- `POST .../monthly-recalculate` — пересчёт одного сотрудника
- `POST /recalculate-all` — массовый пересчёт

**Данные и история:**

- `GET .../stats-history` — история пересчётов (последние 12 мес.)
- `GET /decay-coefficients` — коэффициенты затухания по ролям

## 7.4. Интерфейс пользователя

Платформа предоставляет 4 основных экрана:

### Dashboard — Обзор команды
Командные KPI, графики коммитов за неделю, лента активности, список сотрудников с мини-радарами.

### Profile — Профиль сотрудника
Радарная диаграмма 6 параметров, история пересчётов, кнопка вызова ИИ-рекомендации, детализация по каждому параметру.

### Comparison — Сравнение сотрудников
Наложение радарных диаграмм нескольких сотрудников для визуального выявления skill gaps и дисбалансов.

### Team — Управление командой
Добавление и редактирование сотрудников, массовые операции пересчёта.

## 7.5. Интернационализация

Платформа поддерживает два языка (русский и английский) через систему i18n. Все строки интерфейса хранятся в `assets/js/i18n.js` с использованием `data-i18n` атрибутов в HTML.
